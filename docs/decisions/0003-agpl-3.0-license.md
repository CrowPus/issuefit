# 0003 — AGPL-3.0 license

## Status

Superseded by [0013 — Relicense from AGPL-3.0 to Apache-2.0](0013-apache-2.0-relicense.md)

## Context

IssueFit is an open-source platform that will typically run as a hosted network service.
Permissive licenses (MIT, Apache-2.0) allow anyone to run a modified closed-source copy as a
SaaS without contributing changes back.

## Decision

License the project under **AGPL-3.0-only**. Anyone who runs a modified version of IssueFit as
a network service must offer its users the modified source code.

## Alternatives considered

- **MIT** — maximum adoption and simplicity, but no protection against closed-source hosted
  forks.
- **Apache-2.0** — adds an explicit patent grant but has the same SaaS-fork gap.

## Consequences

- Modifications by anyone hosting IssueFit flow back to the community.
- Some companies avoid AGPL dependencies; this may reduce corporate adoption and
  contributions. This is an accepted trade-off chosen by the project owner.
- All workspace `package.json` files declare `AGPL-3.0-only`; the full text lives in
  `LICENSE`.
